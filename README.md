# Object-Oriented Programming (OOP) in Python 🚀

Welcome to my comprehensive reference repository for mastering Object-Oriented Programming principles in Python. This repository is structured to break down the core pillars and advanced design patterns of OOP into clean, modular scripts.

---

## 📂 Repository Structure

The project code is organized into distinct conceptual directories for easy navigation and focused learning:

### 📁 [01_Basics](./01_Basics/)
Fundamental concepts establishing object construction and state.
*   `classAndObj.py` — Core mechanics of initializing classes and creating operational instances.

### 📁 [02_Encapsulation](./02_Encapsulation/)
Data hiding mechanisms, exposure management, and dynamic attributes.
*   `accessModifiers.py` — Implementations of Public, Protected, and Private variables.
*   `property_demo.py` — Managing attribute access safely using Pythonic getters and setters via `@property`.
*   `Via_Property_computedattributes.py` — Utilizing property decorators to dynamically calculate state attributes.

### 📁 [03_Inheritance](./03_Inheritance/)
Architectural code reuse patterns through class hierarchies.
*   `singleInheritance.py` — Deriving a basic child class behavior from a singular parent.
*   `multiLevel_Inheritance.py` — Grandparent-parent-child structural chains.
*   `multipleInheritance.py` — Deriving from multiple parent classes and dealing with Method Resolution Order (MRO).

### 📁 [04_Polymorphism_and_Abstraction](./04_Polymorphism_and_Abstraction/)
Decoupling interfaces from execution structures.
*   `abstraction.py` — Enforcing architectural blueprints using the `abc` module (`ABC` and `@abstractmethod`).
*   `duckTyping.py` — Dynamic typing principles ("If it walks like a duck...").
*   `methodOverriding.py` — Customizing inherited behavior inside concrete child implementations.

---

## 🛠️ Tech Stack & Concepts Covered
*   **Language:** Python 3.x
*   **Core Concepts:** Encapsulation, Inheritance, Polymorphism, Abstraction, Pythonic Property Decorators, MRO.
